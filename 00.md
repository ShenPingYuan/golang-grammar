# 1. Go 语言简介

Go 是一门面向工程实践的编程语言，其设计目标是让开发者能够更高效地编写可靠、可维护、高性能的软件。

## 1.1 Go 的特点

Go 语言常见特点如下：

- **语法简洁**：语言特性相对克制，学习成本较低
- **编译速度快**：大型项目也能快速构建
- **原生支持并发**：通过 goroutine 和 channel 简化并发编程
- **垃圾回收机制**：减少手动管理内存的复杂度
- **标准库丰富**：网络、文件、编码、测试等开箱即用
- **工程化能力强**：内置格式化、测试、模块管理工具

## 1.2 Go 的应用场景

Go 常用于：

- Web 后端开发
- API 服务开发
- 云原生与容器生态
- 微服务架构
- 网络编程
- 运维工具和命令行工具
- 分布式系统

# 2. Go 开发环境与第一个程序

## 2.1 安装 Go

可以从 Go 官方网站下载安装包并完成安装。安装完成后，执行以下命令检查版本：

```bash
go version
```

如果安装成功，会输出类似信息：

```bash
go version go1.22.0 windows/amd64
```

## 2.2 第一个 Go 程序

创建文件 `main.go`，内容如下：

```go
package main

import "fmt"

func main() {
	fmt.Println("Hello, Go!")
}
```

运行程序：

```bash
go run main.go
```

输出结果：

```bash
Hello, Go!
```

## 2.3 编译 Go 程序

除了直接运行，还可以编译为可执行文件：

```bash
go build main.go
```

编译成功后，会生成可执行文件。

# 3. Go 程序基本结构

Go 程序通常由以下几部分组成：

- 包声明
- 导入包
- 全局声明
- 函数定义
- 程序入口 `main`

## 3.1 package

每个 Go 文件都必须以 `package` 开头，用于声明该文件属于哪个包。

```go
package main
```

如果一个程序要作为可执行程序运行，则必须使用 `main` 包。

## 3.2 import

使用 `import` 导入依赖包。

```go
import "fmt"
```

如果导入多个包，可以使用圆括号：

```go
import (
	"fmt"
	"time"
)
```

## 3.3 main 函数

Go 程序的入口是 `main` 包中的 `main()` 函数。

```go
func main() {
	fmt.Println("程序开始执行")
}
```

## 3.4 注释

Go 支持两种注释方式：

### 单行注释

```go
// 这是单行注释
```

### 多行注释

```go
/*
这是多行注释
可以写多行内容
*/
```

## 3.5 标识符命名规则

标识符包括变量名、函数名、类型名等，命名规则如下：

- 由字母、数字、下划线组成
- 不能以数字开头
- 区分大小写
- 不能与关键字重名

例如：

```go
var name string
var userAge int
var _temp int
```

## 3.6 可见性规则

Go 使用**首字母大小写**控制访问权限：

- **首字母大写**：可被包外访问
- **首字母小写**：仅包内可访问

例如：

```go
type User struct {
	Name string
	age  int
}
```

其中：

- `Name` 可导出
- `age` 不可导出

# 4. 变量与常量

## 4.1 变量定义

Go 中使用 `var` 声明变量。

```go
var name string
var age int
```

也可以声明时直接赋值：

```go
var name string = "Tom"
var age int = 18
```

如果初始化时可以推断类型，则类型可以省略：

```go
var name = "Tom"
var age = 18
```

## 4.2 短变量声明

在函数内部，可以使用 `:=` 快速声明并初始化变量：

```go
name := "Tom"
age := 18
```

注意：

- `:=` 只能在函数内部使用
- 它本质上是“声明并赋值”

## 4.3 多变量声明

```go
var a, b, c int
var x, y = 10, 20
```

也可以分组声明：

```go
var (
	name = "Alice"
	age  = 20
	city = "Beijing"
)
```

## 4.4 匿名变量

Go 使用 `_` 表示匿名变量，用于忽略某个值。

```go
a, _ := 10, 20
fmt.Println(a)
```

常见于函数多返回值场景。

## 4.5 常量

常量使用 `const` 声明，声明后不可修改。

```go
const pi = 3.14159
const site string = "example.com"
```

也可以批量声明：

```go
const (
	A = 1
	B = 2
	C = 3
)
```

## 4.6 iota

`iota` 是 Go 中的常量计数器，常用于定义枚举。

```go
const (
	Monday = iota
	Tuesday
	Wednesday
	Thursday
	Friday
)
```

对应值依次为：

- `Monday = 0`
- `Tuesday = 1`
- `Wednesday = 2`
- `Thursday = 3`
- `Friday = 4`

再看一个例子：

```go
const (
	A = iota // 0
	B        // 1
	C        // 2
)
```

### iota 常见用法

```go
const (
	_  = iota
	KB = 1 << (10 * iota)
	MB
	GB
	TB
)
```

这里可用于表示存储单位。

# 5. 基本数据类型

Go 是静态类型语言，每个变量都有确定类型。

## 5.1 整型

常见整型如下：

- `int`
- `int8`
- `int16`
- `int32`
- `int64`
- `uint`
- `uint8`
- `uint16`
- `uint32`
- `uint64`

示例：

```go
var a int = 10
var b int64 = 100
var c uint = 20
```

## 5.2 浮点型

Go 支持两种浮点类型：

- `float32`
- `float64`

```go
var price float64 = 19.99
```

默认浮点字面量通常被推断为 `float64`。

## 5.3 布尔型

布尔类型只有两个值：

- `true`
- `false`

```go
var ok bool = true
```

注意：Go 中不能将整数直接当作布尔值使用。

## 5.4 字节与字符

- `byte` 是 `uint8` 的别名，常用于表示字节
- `rune` 是 `int32` 的别名，常用于表示 Unicode 字符

```go
var b byte = 'A'
var r rune = '中'
```

## 5.5 字符串

字符串使用双引号包裹：

```go
var s string = "hello"
```

Go 中字符串本质上是**只读字节序列**。

### 转义字符

常见转义字符：

- `\n` 换行
- `\t` 制表符
- `\"` 双引号
- `\\` 反斜杠

```go
fmt.Println("Hello\nGo")
```

### 原始字符串

使用反引号定义原始字符串，保留原始格式：

```go
str := `这是一个原始字符串
不会处理转义字符 \n
可以换行`
```

## 5.6 零值

Go 中变量声明后如果没有显式初始化，会自动赋予零值：

- 数值类型：`0`
- 布尔类型：`false`
- 字符串：`""`
- 指针、切片、map、函数、接口、channel：`nil`

例如：

```go
var a int
var b bool
var c string

fmt.Println(a) // 0
fmt.Println(b) // false
fmt.Println(c) // ""
```

## 5.7 类型转换

Go 不支持隐式类型转换，需要显式转换。

```go
var a int = 10
var b float64 = float64(a)
```

再例如：

```go
var x float64 = 3.14
var y int = int(x)
```

注意：浮点转整数会截断小数部分。

# 6. 运算符

## 6.1 算术运算符

Go 支持常见算术运算符：

- `+`
- `-`
- `*`
- `/`
- `%`
- `++`
- `--`

示例：

```go
a := 10
b := 3

fmt.Println(a + b) // 13
fmt.Println(a - b) // 7
fmt.Println(a * b) // 30
fmt.Println(a / b) // 3
fmt.Println(a % b) // 1
```

注意：

- 整数相除结果仍然是整数
- `++` 和 `--` 在 Go 中是语句，不是表达式

```go
i := 0
i++
```

不能这样写：

```go
// fmt.Println(i++) // 错误
```

## 6.2 比较运算符

- `==`
- `!=`
- `>`
- `<`
- `>=`
- `<=`

```go
fmt.Println(10 > 5)  // true
fmt.Println(10 == 5) // false
```

## 6.3 逻辑运算符

- `&&`
- `||`
- `!`

```go
a := true
b := false

fmt.Println(a && b)
fmt.Println(a || b)
fmt.Println(!a)
```

## 6.4 位运算符

- `&`
- `|`
- `^`
- `<<`
- `>>`

```go
a := 5  // 0101
b := 3  // 0011

fmt.Println(a & b) // 1
fmt.Println(a | b) // 7
fmt.Println(a ^ b) // 6
fmt.Println(a << 1) // 10
fmt.Println(a >> 1) // 2
```

## 6.5 赋值运算符

- `=`
- `+=`
- `-=`
- `*=`
- `/=`
- `%=`

```go
a := 10
a += 5
fmt.Println(a) // 15
```

# 7. 输入输出

Go 常用 `fmt` 包进行输入输出。

## 7.1 输出

### Print

```go
fmt.Print("Hello")
fmt.Print("Go")
```

输出结果：

```bash
HelloGo
```

### Println

```go
fmt.Println("Hello")
fmt.Println("Go")
```

输出结果每次自动换行。

### Printf

```go
name := "Tom"
age := 18
fmt.Printf("name=%s, age=%d\n", name, age)
```

常用格式化占位符：

- `%d`：整数
- `%f`：浮点数
- `%s`：字符串
- `%v`：默认格式
- `%T`：类型
- `%t`：布尔值

## 7.2 输入

### Scan

```go
var name string
var age int
fmt.Scan(&name, &age)
```

### Scanln

```go
var name string
fmt.Scanln(&name)
```

### Scanf

```go
var name string
var age int
fmt.Scanf("%s %d", &name, &age)
```

注意：输入函数需要传入变量地址，因此要使用 `&`。

# 8. 流程控制

## 8.1 if 语句

Go 中 `if` 条件不需要加括号。

```go
age := 20
if age >= 18 {
	fmt.Println("成年人")
}
```

### if else

```go
if age >= 18 {
	fmt.Println("成年人")
} else {
	fmt.Println("未成年人")
}
```

### if 初始化语句

```go
if score := 90; score >= 60 {
	fmt.Println("及格")
}
```

## 8.2 switch 语句

Go 的 `switch` 默认自带 `break`，不需要手动写。

```go
day := 2

switch day {
case 1:
	fmt.Println("Monday")
case 2:
	fmt.Println("Tuesday")
default:
	fmt.Println("Unknown")
}
```

### 多条件匹配

```go
switch day {
case 1, 2, 3, 4, 5:
	fmt.Println("工作日")
case 6, 7:
	fmt.Println("周末")
}
```

### 无表达式 switch

```go
score := 85

switch {
case score >= 90:
	fmt.Println("优秀")
case score >= 60:
	fmt.Println("及格")
default:
	fmt.Println("不及格")
}
```

## 8.3 for 循环

Go 中没有 `while`，统一使用 `for`。

### 标准 for

```go
for i := 0; i < 5; i++ {
	fmt.Println(i)
}
```

### 类 while 写法

```go
i := 0
for i < 5 {
	fmt.Println(i)
	i++
}
```

### 无限循环

```go
for {
	fmt.Println("无限循环")
}
```

## 8.4 range

`range` 常用于遍历数组、切片、map、字符串、channel。

```go
nums := []int{10, 20, 30}
for index, value := range nums {
	fmt.Println(index, value)
}
```

如果不需要索引，可以使用匿名变量：

```go
for _, value := range nums {
	fmt.Println(value)
}
```

## 8.5 break 和 continue

### break

跳出当前循环。

```go
for i := 0; i < 10; i++ {
	if i == 5 {
		break
	}
	fmt.Println(i)
}
```

### continue

跳过本次循环。

```go
for i := 0; i < 5; i++ {
	if i == 2 {
		continue
	}
	fmt.Println(i)
}
```

## 8.6 goto

Go 支持 `goto`，但实际开发中一般不推荐滥用。

```go
i := 0
HERE:
	fmt.Println(i)
	i++
	if i < 3 {
		goto HERE
	}
```

# 9. 数组

数组是长度固定、元素类型相同的数据集合。

## 9.1 数组定义

```go
var arr [3]int
```

表示定义一个长度为 3 的整型数组。

## 9.2 数组初始化

```go
var arr = [3]int{1, 2, 3}
```

也可以让编译器自动推断长度：

```go
arr := [...]int{1, 2, 3, 4}
```

## 9.3 访问和修改数组元素

```go
arr := [3]int{10, 20, 30}
fmt.Println(arr[0])
arr[1] = 99
fmt.Println(arr)
```

## 9.4 遍历数组

### 普通 for

```go
arr := [3]int{1, 2, 3}
for i := 0; i < len(arr); i++ {
	fmt.Println(arr[i])
}
```

### range

```go
for index, value := range arr {
	fmt.Println(index, value)
}
```

## 9.5 数组是值类型

Go 中数组是值类型，赋值会复制整个数组。

```go
a := [3]int{1, 2, 3}
b := a
b[0] = 100

fmt.Println(a) // [1 2 3]
fmt.Println(b) // [100 2 3]
```

这也是为什么在 Go 中更常使用切片而不是数组。

# 10. 切片

切片是对数组的一个连续片段的引用，是 Go 中最常用的数据结构之一。

## 10.1 切片定义

```go
var s []int
```

这定义了一个整型切片，此时它的零值是 `nil`。

## 10.2 基于数组创建切片

```go
arr := [5]int{1, 2, 3, 4, 5}
s := arr[1:4]
fmt.Println(s) // [2 3 4]
```

说明：

- 包含起始索引
- 不包含结束索引

## 10.3 使用字面量创建切片

```go
s := []int{1, 2, 3}
```

## 10.4 使用 make 创建切片

```go
s := make([]int, 3, 5)
```

含义：

- 长度为 3
- 容量为 5

```go
fmt.Println(len(s)) // 3
fmt.Println(cap(s)) // 5
```

## 10.5 append

使用 `append` 向切片追加元素。

```go
s := []int{1, 2}
s = append(s, 3)
s = append(s, 4, 5)
fmt.Println(s)
```

注意：`append` 可能返回新的底层数组，因此必须接收返回值。

## 10.6 copy

`copy` 用于复制切片元素。

```go
src := []int{1, 2, 3}
dst := make([]int, len(src))
copy(dst, src)

fmt.Println(dst)
```

## 10.7 截取切片

```go
s := []int{1, 2, 3, 4, 5}
fmt.Println(s[1:3]) // [2 3]
fmt.Println(s[:3])  // [1 2 3]
fmt.Println(s[2:])  // [3 4 5]
```

## 10.8 切片的本质

切片内部通常包含三个信息：

- 指向底层数组的指针
- 长度 `len`
- 容量 `cap`

因此切片是一个“引用语义”的结构，对切片元素的修改可能影响到底层数组。

```go
arr := [3]int{1, 2, 3}
s := arr[:]
s[0] = 100
fmt.Println(arr) // [100 2 3]
```

# 11. map

`map` 是 Go 中的哈希表结构，用于存储键值对。

## 11.1 map 定义

```go
var m map[string]int
```

此时 `m` 为 `nil`，不能直接写入。

## 11.2 map 初始化

### 使用 make

```go
m := make(map[string]int)
```

### 使用字面量

```go
m := map[string]int{
	"Tom":  18,
	"Jack": 20,
}
```

## 11.3 增删改查

### 添加或修改元素

```go
m := make(map[string]int)
m["Tom"] = 18
m["Tom"] = 20
```

### 读取元素

```go
fmt.Println(m["Tom"])
```

### 删除元素

```go
delete(m, "Tom")
```

## 11.4 判断 key 是否存在

```go
age, ok := m["Tom"]
if ok {
	fmt.Println("存在", age)
} else {
	fmt.Println("不存在")
}
```

这里的 `ok` 是一个布尔值，表示键是否存在。

## 11.5 遍历 map

```go
m := map[string]int{
	"Alice": 18,
	"Bob":   20,
}

for key, value := range m {
	fmt.Println(key, value)
}
```

注意：Go 中 map 遍历顺序是**无序的**。

# 12. 指针

Go 支持指针，但不像 C/C++ 那样复杂，不支持指针运算。

## 12.1 指针的基本概念

指针保存的是变量的内存地址。

```go
a := 10
p := &a

fmt.Println(a)  // 10
fmt.Println(p)  // 地址
fmt.Println(*p) // 10
```

说明：

- `&a`：取地址
- `*p`：根据地址取值

## 12.2 修改指针指向的值

```go
a := 10
p := &a
*p = 20

fmt.Println(a) // 20
```

## 12.3 new 函数

`new` 用于分配内存，并返回对应类型的指针。

```go
p := new(int)
fmt.Println(*p) // 0
*p = 100
fmt.Println(*p) // 100
```

## 12.4 指针作为函数参数

```go
func modify(x *int) {
	*x = 100
}

func main() {
	a := 10
	modify(&a)
	fmt.Println(a) // 100
}
```

当需要在函数内部修改外部变量时，常使用指针参数。

# 13. 函数

函数是 Go 语言组织逻辑的核心。

## 13.1 函数定义

```go
func add(a int, b int) int {
	return a + b
}
```

参数类型相同时，可以简写为：

```go
func add(a, b int) int {
	return a + b
}
```

## 13.2 多返回值

Go 支持一个函数返回多个值。

```go
func calc(a, b int) (int, int) {
	return a + b, a - b
}
```

调用：

```go
sum, diff := calc(10, 5)
fmt.Println(sum, diff)
```

## 13.3 命名返回值

```go
func calc(a, b int) (sum int, diff int) {
	sum = a + b
	diff = a - b
	return
}
```

## 13.4 可变参数

```go
func sum(nums ...int) int {
	total := 0
	for _, v := range nums {
		total += v
	}
	return total
}
```

调用：

```go
fmt.Println(sum(1, 2, 3, 4))
```

## 13.5 匿名函数

```go
func() {
	fmt.Println("这是一个匿名函数")
}()
```

也可以赋值给变量：

```go
add := func(a, b int) int {
	return a + b
}
fmt.Println(add(3, 4))
```

## 13.6 闭包

闭包是引用了外部变量的函数。

```go
func counter() func() int {
	i := 0
	return func() int {
		i++
		return i
	}
}
```

使用：

```go
c := counter()
fmt.Println(c()) // 1
fmt.Println(c()) // 2
fmt.Println(c()) // 3
```

## 13.7 defer

`defer` 用于延迟执行，通常用于资源释放。

```go
func main() {
	defer fmt.Println("最后执行")
	fmt.Println("先执行")
}
```

输出：

```bash
先执行
最后执行
```

### defer 执行顺序

多个 `defer` 按照**后进先出**执行。

```go
defer fmt.Println(1)
defer fmt.Println(2)
defer fmt.Println(3)
```

输出顺序为：

```bash
3
2
1
```

# 14. 结构体

结构体用于将多个不同类型的数据组合成一个整体。

## 14.1 结构体定义

```go
type Person struct {
	Name string
	Age  int
}
```

## 14.2 结构体实例化

### 方式一

```go
var p Person
p.Name = "Tom"
p.Age = 18
```

### 方式二

```go
p := Person{
	Name: "Tom",
	Age:  18,
}
```

### 方式三

```go
p := Person{"Tom", 18}
```

推荐优先使用**带字段名**的写法，可读性更好。

## 14.3 结构体指针

```go
p := &Person{
	Name: "Alice",
	Age:  20,
}
fmt.Println(p.Name)
```

Go 对结构体指针访问字段时可以直接用 `p.Name`，不需要写成 `(*p).Name`。

## 14.4 结构体嵌套

```go
type Address struct {
	City string
}

type User struct {
	Name    string
	Address Address
}
```

使用：

```go
u := User{
	Name: "Tom",
	Address: Address{
		City: "Shanghai",
	},
}
fmt.Println(u.Address.City)
```

## 14.5 匿名字段

```go
type User struct {
	string
	int
}
```

不过这种写法在工程实践中不够直观，通常只在特定场景使用。

# 15. 方法

方法本质上是绑定到某种类型上的函数。

## 15.1 方法定义

```go
type Person struct {
	Name string
}

func (p Person) SayHello() {
	fmt.Println("Hello,", p.Name)
}
```

调用：

```go
p := Person{Name: "Tom"}
p.SayHello()
```

## 15.2 值接收者

```go
func (p Person) SetName(name string) {
	p.Name = name
}
```

这种情况下修改的是副本，不会影响原对象。

## 15.3 指针接收者

```go
func (p *Person) SetName(name string) {
	p.Name = name
}
```

调用：

```go
p := Person{Name: "Tom"}
p.SetName("Jack")
fmt.Println(p.Name) // Jack
```

## 15.4 值接收者与指针接收者区别

### 值接收者

- 接收的是副本
- 修改不会影响原对象
- 适合小对象、只读场景

### 指针接收者

- 接收的是对象地址
- 修改会影响原对象
- 避免大对象拷贝
- 更适合结构体方法

在实际开发中，结构体方法通常更常使用指针接收者。

# 16. 接口

接口是 Go 实现抽象和多态的核心机制。

## 16.1 接口定义

```go
type Speaker interface {
	Speak()
}
```

## 16.2 接口实现

Go 中**不需要显式声明实现某个接口**，只要一个类型实现了接口要求的方法，就自动实现了该接口。

```go
type Dog struct{}

func (d Dog) Speak() {
	fmt.Println("汪汪汪")
}
```

使用：

```go
var s Speaker
s = Dog{}
s.Speak()
```

## 16.3 多态

```go
type Cat struct{}

func (c Cat) Speak() {
	fmt.Println("喵喵喵")
}

func makeSound(s Speaker) {
	s.Speak()
}
```

调用：

```go
makeSound(Dog{})
makeSound(Cat{})
```

## 16.4 空接口

空接口 `interface{}` 可以表示任意类型。

```go
var x interface{}
x = 10
x = "hello"
x = true
```

在 Go 1.18 之前，空接口常用于“通用类型”场景。现在很多场景也可以考虑泛型。

## 16.5 类型断言

```go
var x interface{} = "hello"

str, ok := x.(string)
if ok {
	fmt.Println(str)
}
```

如果断言失败，`ok` 为 `false`。

## 16.6 type switch

```go
func printType(x interface{}) {
	switch v := x.(type) {
	case int:
		fmt.Println("int", v)
	case string:
		fmt.Println("string", v)
	default:
		fmt.Println("unknown")
	}
}
```

# 17. 包、模块与工程化管理

Go 的工程化能力非常强，包管理和模块管理是学习 Go 时必须掌握的内容。

## 17.1 包的基本概念

Go 使用包来组织代码。一个目录通常对应一个包。

例如：

```go
package utils
```

同一个目录下的 Go 文件，通常应该属于同一个包。

## 17.2 自定义包

假设项目结构如下：

```bash
myproject/
├── go.mod
├── main.go
└── utils/
    └── math.go
```

`utils/math.go`：

```go
package utils

func Add(a, b int) int {
	return a + b
}
```

`main.go`：

```go
package main

import (
	"fmt"
	"myproject/utils"
)

func main() {
	fmt.Println(utils.Add(1, 2))
}
```

## 17.3 init 函数

Go 中每个包都可以定义 `init()` 函数，在包初始化时自动执行。

```go
func init() {
	fmt.Println("init 执行")
}
```

特点：

- 不需要手动调用
- 一个包中可以有多个 `init`
- 在 `main()` 执行前运行

## 17.4 go mod

Go 使用模块系统管理依赖。

初始化模块：

```bash
go mod init myproject
```

该命令会生成 `go.mod` 文件。

## 17.5 下载依赖

```bash
go mod tidy
```

该命令会：

- 自动添加需要的依赖
- 删除未使用的依赖
- 更新 `go.sum`

## 17.6 常用命令

```bash
go run .
go build .
go test ./...
go mod tidy
go list ./...
```

## 17.7 包导入规则

导入包时使用模块路径：

```go
import "myproject/utils"
```

注意：

- 导入路径不是文件系统路径，而是模块路径 + 包路径
- 未使用的导入会导致编译错误

## 17.8 Go 工程目录建议

一个简单的 Go 项目结构可以是：

```bash
myproject/
├── go.mod
├── go.sum
├── main.go
├── internal/
├── pkg/
├── api/
├── configs/
└── README.md
```

常见约定：

- `internal/`：仅项目内部使用的包
- `pkg/`：可被外部引用的公共包
- `configs/`：配置文件
- `api/`：接口定义、协议文件等

# 18. 错误处理

Go 没有传统的异常机制作为主要错误处理手段，而是倾向于显式返回错误值。

## 18.1 error 接口

Go 内置 `error` 接口：

```go
type error interface {
	Error() string
}
```

## 18.2 返回错误

```go
func divide(a, b float64) (float64, error) {
	if b == 0 {
		return 0, fmt.Errorf("除数不能为 0")
	}
	return a / b, nil
}
```

调用：

```go
result, err := divide(10, 0)
if err != nil {
	fmt.Println("错误:", err)
	return
}
fmt.Println(result)
```

## 18.3 errors.New

```go
import "errors"

func checkAge(age int) error {
	if age < 0 {
		return errors.New("年龄不能为负数")
	}
	return nil
}
```

## 18.4 fmt.Errorf

```go
return fmt.Errorf("读取文件失败: %v", err)
```

## 18.5 panic

`panic` 表示程序发生严重错误，通常会中断程序执行。

```go
panic("发生严重错误")
```

适用场景：

- 不可恢复的错误
- 程序初始化失败
- 断言某些绝不应该发生的情况

## 18.6 recover

`recover` 可以捕获 `panic`，防止程序直接崩溃，但必须配合 `defer` 使用。

```go
func test() {
	defer func() {
		if err := recover(); err != nil {
			fmt.Println("捕获到 panic:", err)
		}
	}()

	panic("出错了")
}
```

一般业务错误不要滥用 `panic`，优先返回 `error`。

# 19. 并发编程

Go 的并发编程是其最具代表性的特性之一。

## 19.1 goroutine

goroutine 是 Go 中的轻量级线程。

```go
func task() {
	fmt.Println("执行任务")
}

func main() {
	go task()
	fmt.Println("main")
}
```

注意：主 goroutine 结束后，其他 goroutine 也可能来不及执行，因此实际中通常需要同步机制。

## 19.2 channel

channel 用于 goroutine 之间通信。

### 创建 channel

```go
ch := make(chan int)
```

### 发送数据

```go
ch <- 10
```

### 接收数据

```go
x := <-ch
fmt.Println(x)
```

## 19.3 使用 goroutine + channel

```go
func main() {
	ch := make(chan int)

	go func() {
		ch <- 100
	}()

	value := <-ch
	fmt.Println(value)
}
```

## 19.4 无缓冲 channel

```go
ch := make(chan int)
```

无缓冲 channel 的发送和接收必须同时准备好，否则会阻塞。

## 19.5 有缓冲 channel

```go
ch := make(chan int, 2)
ch <- 1
ch <- 2
```

带缓冲区的 channel 在缓冲区未满前，发送不会阻塞。

## 19.6 关闭 channel

```go
close(ch)
```

关闭后不能继续发送数据，但仍可以继续接收已经存在的数据。

## 19.7 range 遍历 channel

```go
ch := make(chan int, 3)
ch <- 1
ch <- 2
ch <- 3
close(ch)

for v := range ch {
	fmt.Println(v)
}
```

## 19.8 select

`select` 用于同时监听多个 channel。

```go
select {
case msg := <-ch1:
	fmt.Println("来自 ch1:", msg)
case msg := <-ch2:
	fmt.Println("来自 ch2:", msg)
default:
	fmt.Println("没有数据")
}
```

## 19.9 sync.WaitGroup

`WaitGroup` 用于等待一组 goroutine 执行完成。

```go
package main

import (
	"fmt"
	"sync"
)

func worker(id int, wg *sync.WaitGroup) {
	defer wg.Done()
	fmt.Println("worker", id)
}

func main() {
	var wg sync.WaitGroup

	for i := 1; i <= 3; i++ {
		wg.Add(1)
		go worker(i, &wg)
	}

	wg.Wait()
	fmt.Println("所有任务完成")
}
```

## 19.10 互斥锁 sync.Mutex

当多个 goroutine 同时访问共享资源时，需要使用互斥锁。

```go
package main

import (
	"fmt"
	"sync"
)

func main() {
	var mu sync.Mutex
	var count int
	var wg sync.WaitGroup

	for i := 0; i < 1000; i++ {
		wg.Add(1)
		go func() {
			defer wg.Done()
			mu.Lock()
			count++
			mu.Unlock()
		}()
	}

	wg.Wait()
	fmt.Println(count)
}
```

# 22. Go 常见易错点总结

## 22.1 已声明但未使用

Go 对未使用变量和未使用导入非常严格。

```go
x := 10 // 如果不用，会编译报错
```

这是 Go 强调代码整洁的一种体现。

## 22.2 map 未初始化不能直接赋值

错误示例：

```go
var m map[string]int
m["a"] = 1
```

正确写法：

```go
m := make(map[string]int)
m["a"] = 1
```

## 22.3 切片 append 后可能指向新底层数组

```go
s := []int{1, 2, 3}
s = append(s, 4)
```

如果底层数组容量不足，`append` 会分配新数组，因此必须重新接收返回值。

## 22.4 数组是值类型，切片是引用语义

数组赋值是整份复制，切片赋值共享底层数据，这一点非常容易混淆。

## 22.5 defer 参数求值时机

```go
x := 1
defer fmt.Println(x)
x = 2
```

输出为：

```bash
1
```

因为 `defer` 在声明时就已经确定参数值。

## 22.6 range 变量问题

在某些循环场景中，`range` 中的迭代变量是复用的，闭包引用时要特别小心。写并发代码时尤其容易踩坑。

例如应尽量这样写：

```go
for _, v := range nums {
	v := v
	go func() {
		fmt.Println(v)
	}()
}
```

## 22.7 nil 问题

以下类型可能为 `nil`：

- 指针
- 切片
- map
- channel
- 函数
- 接口

使用前要注意是否已经初始化。

## 22.8 接口值不等于 nil 的陷阱

一个接口如果内部持有“带类型的 nil 指针”，接口本身不一定等于 `nil`。这是 Go 中较经典的坑，实际开发中需要谨慎判断。
